🔍 LibreChat转发日志测试

功能说明：
现在已经为所有AI模型添加了详细的转发日志，你可以在服务器日志中看到完整的请求转发过程。
📋 日志类型说明
🚀[OpenAI转发配置] - 客户端初始化时的配置信息
📤[OpenAI转发配置] - 客户端选项详情
🌐[OpenAI请求转发] - 准备发送请求的详细信息
💬[OpenAI请求转发] - 消息内容（脱敏处理）
🔧[OpenAI客户端创建] - OpenAI客户端创建参数
📡[OpenAI请求发送] - 即将发送到上游服务
🌊[OpenAI流式请求] - 流式请求的详细信息
✅[OpenAI响应接收] - 收到上游服务响应
❌[OpenAI请求失败] - 请求失败时的错误信息
🚀[Anthropic转发配置] - Anthropic客户端配置
📡[Anthropic请求发送] - Anthropic请求详情
✅[Anthropic响应接收] - Anthropic响应信息

🔍 如何查看转发日志：

  1. 启动LibreChat服务：确保服务正在运行
  2. 查看服务器日志：
    • 如果使用Docker：docker logs librechat
    • 如果直接运行：查看控制台输出
    • 如果使用PM2：pm2 logs
  3. 登录并发送消息：使用外部认证登录并发送AI消息
  4. 观察日志输出：在日志中查找带有emoji的转发日志

📊 日志包含的信息：

请求信息
  • 目标URL（应该是 https://api-dev.718ai.cn/v1/...）
  • 使用的模型名称
  • API Key前缀（脱敏显示）
  • 请求参数（温度、最大token等）
  • 消息数量和内容预览
  • 用户ID和时间戳
响应信息
  • 响应时间
  • 返回的模型信息
  • Token使用情况
  • 错误信息（如果有）
  • 流式响应的处理状态

⚠️ 注意事项：

  • 日志中的API Key已经脱敏处理，只显示前10个字符
  • 消息内容只显示前100个字符的预览
  • 所有敏感信息都经过了安全处理
  • 日志级别设置为INFO，确保能看到转发日志
